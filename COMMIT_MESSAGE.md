## 구조
```
<type>: <subject> `<file name | function name>`
```

## 규칙

### type
`fix` : 버그 수정 / 코드 수정

`refactor` : 코드 리팩토링

`feat` : 기능 추가

`comment` : 주석 추가

<br>

### subject
소문자만 사용해주세요

문장 끝에 마침표를 넣지 마세요

**한국어를 사용해도 되지만 영어를 사용하는걸 권장합니다**

<br>


### file name | function name
파일명이나 함수 이름 등을 지칭하는 경우 대문자 사용이 가능하며 ` 로 감싸서 표현합니다.

<br>

## 예시
```
fix: edit header margin `header.css`
```
